# Memory Taxonomy

Memory is allowed only when reviewed.

## Types

- Useful pattern: repeatable design that worked.
- Failed idea: idea rejected after evidence, prototype, or production use.
- Strong evidence source: source class that proved reliable in a domain.
- Reusable architecture: implementation pattern that reduced risk or toil.
- Dangerous assumption: belief that caused or could cause poor decisions.
- Validation result: measured outcome from test, pilot, rollout, or review.
- Domain-specific constraint: local rule, workflow, regulation, or integration boundary.

## Retention

- Review memories tied to changing APIs, regulations, products, prices, or model behavior.
- Keep domain and applicability narrow.
- Store counterexamples with every broad insight.
