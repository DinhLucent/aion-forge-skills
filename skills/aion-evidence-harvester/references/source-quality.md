# Source Quality

Rank evidence before using it.

- Strong: official docs, standards, regulations, internal incidents, support tickets, reproducible logs, peer-reviewed papers, mature postmortems.
- Medium: GitHub maintainer comments, release notes, changelogs, architecture records, competitor docs.
- Weak: marketing pages, social posts, forum anecdotes, unverified complaints.

Use weak sources to form questions. Use strong or converging medium sources for build recommendations.

Always record date/version, limitation, and confidence.
