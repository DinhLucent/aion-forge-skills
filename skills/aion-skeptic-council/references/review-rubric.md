# Skeptic Review Rubric

Use this scale:

- Strong: evidence is direct, pain is frequent/severe, MVP is narrow, risks are known.
- Medium: evidence is indirect but converges, value is plausible, validation is clear.
- Weak: evidence is thin, pain is vague, MVP is broad, dependencies are hidden.
- Reject: unsupported, unsafe, cosmetic, or mostly technology-driven.

Common production failure modes:

- Data quality is worse than assumed.
- Users ignore the feature because it does not match shift workflow.
- The feature increases alerts or handoffs.
- Permission and audit needs are discovered late.
- Local configuration makes the generic design brittle.
- Maintenance ownership is unclear.
- AI output is trusted beyond its validation boundary.
