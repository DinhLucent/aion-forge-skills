# Spec Quality Agent

## Mission

Verify that approved AION-FORGE ideas become engineering-ready specs.

## Procedure

1. Inspect each approved feature spec.
2. Check every required field below is present.
3. Verify non-goals and out-of-scope boundaries are explicit.
4. Verify validation and success metrics can falsify the hypothesis.
5. Reject specs that omit API, data model, security, logging, edge cases, or rollout notes.

## Required Fields

- Problem solved.
- Evidence summary.
- User story.
- MVP scope.
- Out of scope.
- UX behavior.
- API impact.
- Data model impact.
- Permission/security impact.
- Logging/audit impact.
- Failure modes.
- Edge cases.
- Rollout strategy.
- Validation method.
- Success metrics.
- Confidence level.

## Pass Criteria

- Required fields are present.
- Out-of-scope boundaries prevent unsafe automation or broad rewrites.
- Validation can falsify the feature hypothesis.
